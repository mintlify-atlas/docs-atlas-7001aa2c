This guide covers everything you need to install and configure ZOODOM for local development or production deployment.

## System Requirements

### Required Software

<AccordionGroup>
  <Accordion title="Node.js >= 22.12.0">
    ZOODOM requires Node.js version 22.12.0 or higher as specified in `package.json`:

    ```json package.json
    "engines": {
      "node": ">=22.12.0"
    }
    ```

    **Installation:**
    - Download from [nodejs.org](https://nodejs.org/)
    - Or use a version manager like [nvm](https://github.com/nvm-sh/nvm):
      ```bash
      nvm install 22.12.0
      nvm use 22.12.0
      ```

    **Verify installation:**
    ```bash
    node --version
    # Should output v22.12.0 or higher
    ```
  </Accordion>

  <Accordion title="npm Package Manager">
    npm comes bundled with Node.js. Verify you have npm installed:

    ```bash
    npm --version
    ```

    Update npm to the latest version if needed:
    ```bash
    npm install -g npm@latest
    ```
  </Accordion>

  <Accordion title="Git Version Control">
    Required for cloning the repository and version control.

    **Installation:**
    - Windows: [git-scm.com](https://git-scm.com/)
    - macOS: `brew install git` or Xcode Command Line Tools
    - Linux: `sudo apt install git` or `sudo yum install git`

    **Verify installation:**
    ```bash
    git --version
    ```
  </Accordion>
</AccordionGroup>

### Recommended Tools

- **Code Editor**: VS Code with Astro extension
- **Browser**: Modern browser with DevTools (Chrome, Firefox, Edge, Safari)
- **Terminal**: Any modern terminal (integrated terminal in VS Code works great)

## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    Get the source code from your repository:

    ```bash
    git clone <repository-url>
    cd zoodom
    ```

    This will create a `zoodom` directory with all project files.
  </Step>

  <Step title="Install Dependencies">
    Install all required npm packages:

    ```bash
    npm install
    ```

    This installs the dependencies defined in `package.json`:

    ```json package.json
    "dependencies": {
      "astro": "^6.0.4"
    }
    ```

    <Note>
    The installation downloads Astro and its peer dependencies. Bootstrap is loaded via CDN, so it's not in the dependencies list.
    </Note>
  </Step>

  <Step title="Verify Installation">
    Check that everything installed correctly:

    ```bash
    npm run astro -- --version
    ```

    This should output the Astro version (6.0.4 or compatible).
  </Step>

  <Step title="Start Development Server">
    Launch the local development server:

    ```bash
    npm run dev
    ```

    Expected output:
    ```
    🚀 astro v6.0.4 started in XXms

    ┃ Local    http://localhost:4321/
    ┃ Network  use --host to expose
    ```
  </Step>

  <Step title="Access the Application">
    Open your browser and navigate to:

    ```
    http://localhost:4321
    ```

    You should see the ZOODOM homepage.
  </Step>
</Steps>

## Configuration

### Astro Configuration

The project configuration is defined in `astro.config.mjs`:

```javascript astro.config.mjs
// @ts-check
import { defineConfig } from 'astro/config';

// https://astro.build/config
export default defineConfig({
    site: 'https://zoodomi.netlify.app',
});
```

<ParamField path="site" type="string" default="https://zoodomi.netlify.app">
  The production URL of your site. Used for generating canonical URLs and sitemaps.
</ParamField>

### TypeScript Configuration

ZOODOM uses TypeScript with Astro's strict configuration:

```json tsconfig.json
{
  "extends": "astro/tsconfigs/strict",
  "include": [".astro/types.d.ts", "**/*"],
  "exclude": ["dist"]
}
```

This provides type safety for:
- Astro components (`.astro` files)
- Component props and interfaces
- Asset imports

### SEO and Meta Tags

The Head component (`src/components/Head.astro`) provides comprehensive SEO configuration:

```typescript src/components/Head.astro
interface Props {
    title?: string;
    description?: string;
    canonicalUrl?: string;
    image?: string;
}

const {
    title = 'ZOODOM - Parque Zoológico Nacional',
    description = 'ZOODOM - Parque Zoológico Nacional. Descubre la fauna y flora en nuestro zoológico.',
    canonicalUrl = 'https://zoodomi.netlify.app',
    image = '/cover.png',

```

Features:
- Open Graph tags for social media sharing
- Meta description and keywords for SEO
- Canonical URLs
- Spanish language optimization (`lang="es"`, `og:locale="es_DO"`)
- Google Fonts integration (Onest font family)

## Building for Production

### Build Command

Create an optimized production build:

```bash
npm run build
```

This command:
1. Compiles all `.astro` files to static HTML
2. Optimizes images and assets
3. Generates static files in the `./dist` directory
4. Creates a production-ready bundle

### Preview Build Locally

Test the production build before deploying:

```bash
npm run preview
```

This serves the `dist` folder locally, allowing you to verify the production build.

### Deployment to Netlify

ZOODOM is configured for deployment on Netlify at https://zoodomi.netlify.app

**Build settings:**
- **Build command**: `npm run build`
- **Publish directory**: `dist`
- **Node version**: 22.12.0 or higher

<Note>
Netlify automatically detects Astro projects and applies optimal build settings. No additional configuration file is needed.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Error: Node version is too old">
    **Problem**: You're running an older version of Node.js

    **Solution**: Upgrade to Node.js 22.12.0 or higher:
    ```bash
    # Using nvm
    nvm install 22.12.0
    nvm use 22.12.0

    # Or download from nodejs.org
    ```

    Verify the version:
    ```bash
    node --version
    ```
  </Accordion>

  <Accordion title="Error: npm install fails">
    **Problem**: Dependency installation errors

    **Solutions**:

    1. Clear npm cache:
    ```bash
    npm cache clean --force
    rm -rf node_modules package-lock.json
    npm install
    ```

    2. Check your npm version:
    ```bash
    npm --version
    npm install -g npm@latest
    ```

    3. Verify you have sufficient disk space and permissions
  </Accordion>

  <Accordion title="Error: Port 4321 already in use">
    **Problem**: Another service is using port 4321

    **Solutions**:

    1. Use a different port:
    ```bash
    npm run dev -- --port 3000
    ```

    2. Kill the process using port 4321:
    ```bash
    # On macOS/Linux
    lsof -ti:4321 | xargs kill -9

    # On Windows
    netstat -ano | findstr :4321
    taskkill /PID <PID> /F
    ```
  </Accordion>

  <Accordion title="Images not loading">
    **Problem**: Images in `src/assets/` not displaying

    **Solution**: Astro requires importing images in `.astro` files:

    ```astro
    ---
    import logo from '../assets/Logo.svg';
    import HeroImg from '../assets/hero-img.png';
    ---

    <img src={logo.src} alt="ZOODOM Logo">
    <img src={HeroImg.src} alt="Hero Image">
    ```

    Static files in the `public/` directory can be referenced directly:
    ```html
    <img src="/cover.png" alt="Cover">
    ```
  </Accordion>

  <Accordion title="CSS styles not applying">
    **Problem**: Custom styles from `global.css` not working

    **Solution**: Ensure global.css is imported in the Head component:

    ```astro src/components/Head.astro
    ---
    import '../assets/global.css';
    ---
    ```

    The global.css file defines CSS custom properties (variables) that must be loaded to use the design system colors.
  </Accordion>

  <Accordion title="Bootstrap components not working">
    **Problem**: Bootstrap JavaScript features (modals, dropdowns) not functioning

    **Solution**: Verify all Bootstrap scripts are loaded in the Layout component:

    ```astro src/layouts/Layout.astro
    <!-- Popper.js (required for Bootstrap) -->
    <script src="https://cdn.jsdelivr.net/npm/@popperjs/core@2.11.8/dist/umd/popper.min.js"></script>
    
    <!-- Bootstrap JS -->
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.min.js"></script>
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.bundle.min.js"></script>
    ```
  </Accordion>
</AccordionGroup>
