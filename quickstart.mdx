This guide will help you get the ZOODOM zoo website running locally on your machine in just a few minutes.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** version 22.12.0 or higher
- **npm** (comes with Node.js)
- **Git** for cloning the repository

<Tip>
You can check your Node.js version by running `node --version` in your terminal.
</Tip>

## Quick Setup

<Steps>
  <Step title="Clone the repository">
    Clone the ZOODOM project to your local machine:

    ```bash
    git clone <repository-url>
    cd zoodom
    ```
  </Step>

  <Step title="Install dependencies">
    Install the required npm packages:

    ```bash
    npm install
    ```

    This will install:
    - Astro 6.0.4 (the static site framework)
    - All required dependencies

    <Note>
    ZOODOM uses Bootstrap 5.3.8 via CDN, so no additional CSS framework installation is needed.
    </Note>
  </Step>

  <Step title="Start the development server">
    Launch the local development server:

    ```bash
    npm run dev
    ```

    The site will be available at `http://localhost:4321`
  </Step>

  <Step title="View the site">
    Open your browser and navigate to:

    ```
    http://localhost:4321
    ```

    You should see the ZOODOM homepage with:
    - Navigation bar with links to Inicio, Especies, Galeria, Blog, and Contacto
    - Hero section with "Bienvenido a nuestro zoológico" heading
    - Call-to-action button for reservations
  </Step>
</Steps>

## Project Structure

Once you have the project running, here's what you'll find:

```text
zoodom/
├── public/              # Static assets
│   └── cover.png       # Social media cover image
├── src/
│   ├── assets/         # Images and styles
│   │   ├── Logo.svg    # ZOODOM logo
│   │   ├── hero-img.png # Hero section image
│   │   ├── Texture.png  # Texture backgrounds
│   │   └── global.css   # Global styles and CSS variables
│   ├── components/     # Reusable Astro components
│   │   └── Head.astro   # SEO and meta tags
│   ├── layouts/        # Layout templates
│   │   └── Layout.astro # Main layout with Bootstrap
│   └── pages/          # Routes (file-based routing)
│       └── index.astro  # Homepage
├── astro.config.mjs    # Astro configuration
├── package.json        # Dependencies and scripts
└── tsconfig.json       # TypeScript configuration
```

## Available Commands

All commands are run from the root of the project:

| Command | Action |
| :--- | :--- |
| `npm install` | Installs dependencies |
| `npm run dev` | Starts local dev server at `localhost:4321` |
| `npm run build` | Build your production site to `./dist/` |
| `npm run preview` | Preview your build locally, before deploying |
| `npm run astro ...` | Run CLI commands like `astro add`, `astro check` |

## Understanding the Tech Stack

### Astro 6.0.4

ZOODOM is built with Astro, a modern static site generator that:
- Ships zero JavaScript by default for better performance
- Uses file-based routing (files in `src/pages/` become routes)
- Supports component-based architecture with `.astro` files

### Bootstrap 5.3.8

The UI is built with Bootstrap, loaded via CDN in the Layout component:

```astro src/layouts/Layout.astro
<link
  href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css"
  rel="stylesheet"
  integrity="sha384-sRIl4kxILFvY47J16cr9ZwB07vP4J8+LH7qKQnuqkuIAvNWLzeN8tE5YBujZqJLB"
  crossorigin="anonymous"
/>
```

### Custom Design System

ZOODOM includes a custom CSS design system with nature-inspired colors defined in `src/assets/global.css`:

- **Brand Green**: Primary zoo theme colors (from `#144A1F` to `#F4FFF6`)
- **Accent Green**: Secondary nature tones (from `#2E4F19` to `#F7FFF3`)
- **Extended Colors**: Blue, yellow, orange, and red for accents
- **Neutral Grays**: Grayscale palette for text and backgrounds
