## Overview

ZOODOM organizes assets into two main locations: the `src/assets/` directory for processed assets and the `public/` directory for static files served directly.

## Asset Directories

```
src/assets/          → Processed by Astro build pipeline
├── Logo.svg         → ZOODOM logo (29 KB)
├── hero-img.png     → Homepage hero image (853 KB)
├── Texture.png      → Decorative texture (1.5 MB)
└── global.css       → Global styles and design system

public/              → Served directly without processing
└── cover.png        → Social media cover image (670 KB)
```

## Source Assets vs Public Assets

<CardGroup cols={2}>
  <Card title="src/assets/" icon="wand-magic-sparkles">
    **Processed Assets**
    
    - Optimized during build
    - Hashed filenames for caching
    - Import in components
    - Recommended for images in content
  </Card>
  
  <Card title="public/" icon="globe">
    **Static Assets**
    
    - Served directly, no processing
    - Keep original filenames
    - Reference with absolute paths
    - Good for SEO files, robots.txt, favicons
  </Card>
</CardGroup>

## Images

### Logo (Logo.svg)

<Tabs>
  <Tab title="Details">
    - **Format:** SVG (Scalable Vector Graphics)
    - **Size:** 29 KB
    - **Usage:** Navigation bar branding
    - **Location:** `src/assets/Logo.svg`
  </Tab>
  
  <Tab title="Usage in Code">
    ```astro src/pages/index.astro
    ---
    import logo from '../assets/logo.svg';
    ---
    
    <a class="navbar-brand" href="#">
      <img src={logo.src} class="logo" alt="">
    </a>
    ```
    
    **CSS Styling:**
    ```css src/assets/global.css
    .logo {
        width: 110px;
        height: auto;
    }
    ```
  </Tab>
  
  <Tab title="Why SVG?">
    **Benefits of SVG logos:**
    - Scales perfectly at any size
    - Small file size (29 KB)
    - Crisp on retina displays
    - Can be styled with CSS
    - No pixelation when zoomed
  </Tab>
</Tabs>

### Hero Image (hero-img.png)

<Tabs>
  <Tab title="Details">
    - **Format:** PNG
    - **Size:** 853 KB
    - **Usage:** Homepage hero section
    - **Location:** `src/assets/hero-img.png`
    - **Positioning:** Absolutely positioned in layout
  </Tab>
  
  <Tab title="Usage in Code">
    ```astro src/pages/index.astro
    ---
    import HeroImg from '../assets/hero-img.png';
    ---
    
    <section class="col-lg-6 col-md-6 col-12 position-relative">
      <img src={HeroImg.src} class="img-fluid position-absolute" alt="Hero Image">
    </section>
    ```
    
    **Bootstrap Classes:**
    - `img-fluid` - Responsive scaling
    - `position-absolute` - Absolute positioning
  </Tab>
  
  <Tab title="Optimization">
    <Warning>
      **Large File Size:** At 853 KB, this image could be optimized:
      
      - Convert to WebP format (50-80% smaller)
      - Use responsive images with `srcset`
      - Lazy load below the fold
      - Compress with tools like TinyPNG
    </Warning>
  </Tab>
</Tabs>

### Texture Image (Texture.png)

<Tabs>
  <Tab title="Details">
    - **Format:** PNG
    - **Size:** 1.5 MB
    - **Usage:** Decorative background element
    - **Location:** `src/assets/Texture.png`
  </Tab>
  
  <Tab title="Usage in Code">
    ```astro src/pages/index.astro
    ---
    import Texture from '../assets/texture.png';
    ---
    
    <section id="">
      <img src={Texture.src} alt="Texture" class="img-fluid">
      <section class="container">
      </section>
    </section>
    ```
  </Tab>
  
  <Tab title="Optimization">
    <Warning>
      **Very Large File:** At 1.5 MB, this is a performance concern:
      
      **Recommendations:**
      - Use as CSS background instead
      - Convert to WebP or AVIF
      - Apply aggressive compression
      - Consider CSS patterns instead
      - Use `loading="lazy"` attribute
    </Warning>
  </Tab>
</Tabs>

### Cover Image (cover.png)

<Tabs>
  <Tab title="Details">
    - **Format:** PNG
    - **Size:** 670 KB
    - **Usage:** Open Graph social media preview
    - **Location:** `public/cover.png`
    - **Dimensions:** Should be 1200×630 px for optimal social sharing
  </Tab>
  
  <Tab title="Usage in Code">
    ```astro src/components/Head.astro
    ---
    const {
      image = '/cover.png',
      canonicalUrl = 'https://zoodomi.netlify.app'
    } = Astro.props;
    
    const socialImageUrl = new URL(image, canonicalUrl).href;
    ---
    
    <meta property="og:image" content={socialImageUrl} />
    <meta property="og:image:width" content="1200" />
    <meta property="og:image:height" content="630" />
    ```
  </Tab>
  
  <Tab title="Social Media">
    **Why public/?**
    
    Social media crawlers need a direct, unchanging URL. Files in `public/` maintain their paths and aren't processed, making them ideal for:
    
    - Open Graph images
    - Favicons
    - robots.txt
    - sitemap.xml
  </Tab>
</Tabs>

<Note>

#### Global Resets

```css src/assets/global.css
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}
```

**This reset ensures:**
- Consistent spacing across browsers
- Predictable box model behavior
- No unexpected margins or padding

## Asset Import Patterns

### Importing Images in Astro

<CodeGroup>
```astro Method 1: Direct Import
---
import logo from '../assets/logo.svg';
import heroImg from '../assets/hero-img.png';
---

<img src={logo.src} alt="Logo" />
<img src={heroImg.src} alt="Hero" />
```

```astro Method 2: Import Object
---
import * as assets from '../assets/*.{png,svg}';
---

<img src={assets.logo.src} alt="Logo" />
```

```astro Method 3: Public Assets
<!-- No import needed for public/ files -->
<img src="/cover.png" alt="Cover" />
```
</CodeGroup>

### Importing Styles

```astro src/components/Head.astro
---
import '../assets/global.css';
---
```

<Check>
  Importing CSS in a component makes it available globally throughout your application.
</Check>

## Asset Organization Best Practices

<CardGroup cols={2}>
  <Card title="Use src/assets/ for" icon="wand-sparkles">
    - Component images
    - Hero images
    - Icons and illustrations
    - CSS/SCSS files
    - Images that need optimization
  </Card>
  
  <Card title="Use public/ for" icon="folder">
    - favicon.ico
    - robots.txt
    - sitemap.xml
    - Social media images
    - Files that need consistent URLs
  </Card>
  
  <Card title="Format Guidelines" icon="image">
    - **SVG** for logos and icons
    - **WebP** for photos (with PNG fallback)
    - **PNG** for images needing transparency
    - **JPG** for photographs without transparency
  </Card>
  
  <Card title="Naming Conventions" icon="tag">
    - Use kebab-case: `hero-img.png`
    - Be descriptive: `animal-card-bg.png`
    - Include dimensions if multiple sizes: `logo-small.svg`
  </Card>
</CardGroup>

## Adding New Assets

<AccordionGroup>
  <Accordion title="Adding Images">
    1. **Place file in appropriate directory:**
       ```bash
       src/assets/        # For optimized assets
       public/           # For static assets
       ```
    
    2. **Import in your component:**
       ```astro
       ---
       import myImage from '../assets/my-image.png';
       ---
       
       <img src={myImage.src} alt="Description" />
       ```
    
    3. **Or reference public assets directly:**
       ```html
       <img src="/my-static-image.png" alt="Description" />
       ```
  </Accordion>
  
  <Accordion title="Adding Fonts">
    **Option 1: Google Fonts (Current method)**
    ```html src/components/Head.astro
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
    <link href="https://fonts.googleapis.com/css2?family=Onest:wght@100..900&display=swap" rel="stylesheet">
    ```
    
    **Option 2: Self-hosted fonts**
    ```css
    @font-face {
      font-family: 'Onest';
      src: url('/fonts/Onest-Variable.woff2') format('woff2');
      font-weight: 100 900;
      font-display: swap;
    }
    ```
  </Accordion>
  
  <Accordion title="Adding Icons">
    **Option 1: SVG Icons**
    ```astro
    ---
    import iconSearch from '../assets/icons/search.svg';
    ---
    
    <img src={iconSearch.src} alt="Search" class="icon" />
    ```
    
    **Option 2: Icon Library (recommended)**
    ```bash
    npm install @astrojs/icon
    ```
    
    ```astro
    ---
    import { Icon } from 'astro-icon';
    ---
    
    <Icon name="mdi:magnify" />
    ```
  </Accordion>
</AccordionGroup>
