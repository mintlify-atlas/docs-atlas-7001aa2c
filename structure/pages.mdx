## Overview

ZOODOM uses **Astro's file-based routing** system. Every `.astro` file in the `src/pages/` directory automatically becomes a route in your website.

## Current Pages

The project currently has one main page:

```
src/pages/
└── index.astro    → Routes to / (homepage)
```

<Info>
  **File-based Routing:** In Astro, the file name determines the URL path. For example, `about.astro` would create the `/about` route.
</Info>

## Homepage Structure

The homepage (`index.astro`) is the main entry point for visitors. Let's break down its structure:

### Frontmatter

The frontmatter section imports dependencies:

```astro src/pages/index.astro
---
import Layout from '../layouts/Layout.astro';
import logo from'../assets/logo.svg';
import HeroImg from '../assets/hero-img.png';
import Texture from '../assets/texture.png';
---
```

<Note>
  Astro's asset imports are automatically optimized during the build process for better performance.
</Note>

## Layout Wrapper

The entire page is wrapped in the `Layout` component:

```astro src/pages/index.astro
<Layout>
  <main>
    <!-- Page content here -->
  </main>
</Layout>
```

<Check>
  This pattern ensures consistent HTML structure, meta tags, and global scripts across all pages.
</Check>

## Adding New Pages

To add a new page to ZOODOM:

<Steps>
  <Step title="Create a new .astro file">
    Add a new file in `src/pages/`. For example, `especies.astro` for `/especies` route.
  </Step>
  
  <Step title="Import the Layout">
    ```astro
    ---
    import Layout from '../layouts/Layout.astro';
    ---
    ```
  </Step>
  
  <Step title="Wrap content in Layout">
    ```astro
    <Layout>
      <main>
        <!-- Your page content -->
      </main>
    </Layout>
    ```
  </Step>
  
  <Step title="Update navigation links">
    Update the href attributes in `index.astro` navigation to link to your new pages.
  </Step>
</Steps>

<Warning>
  **Important:** Currently, all navigation links point to `#` (same page). Update these to actual routes like `/especies`, `/galeria`, etc. when creating new pages.
</Warning>

## Best Practices

<CardGroup cols={2}>
  <Card title="Semantic HTML" icon="code">
    Use appropriate HTML5 elements like `<nav>`, `<header>`, `<main>`, and `<section>`
  </Card>
  
  <Card title="Accessibility" icon="universal-access">
    Include alt text for images and ARIA attributes for navigation
  </Card>
  
  <Card title="Asset Imports" icon="file-import">
    Import assets in frontmatter for Astro's optimization pipeline
  </Card>
  
  <Card title="Layout Consistency" icon="layer-group">
    Always wrap pages in the Layout component for consistent structure
  </Card>
</CardGroup>
