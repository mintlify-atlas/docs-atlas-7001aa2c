## Introduction

ZOODOM is a zoo website built with **Astro**, a modern web framework that delivers fast, content-focused websites. The project follows a clean, organized structure that separates concerns between pages, components, layouts, and assets.

## Directory Structure

The ZOODOM project is organized into the following key directories:

<CodeGroup>
```bash Project Tree
src/
├── assets/          # Images, styles, and static resources
│   ├── Logo.svg
│   ├── hero-img.png
│   ├── Texture.png
│   └── global.css
├── components/      # Reusable Astro components
│   └── Head.astro
├── layouts/         # Page layout templates
│   └── Layout.astro
└── pages/          # File-based routing pages
    └── index.astro

public/             # Static assets served directly
└── cover.png
```
</CodeGroup>

## Technology Stack

## Design System

The project uses a comprehensive color system defined in CSS custom properties:

<Tabs>
  <Tab title="Brand Colors">
    **Green Palette** - Primary brand colors ranging from `green-50` to `green-900`
    
    Used for primary actions, branding elements, and success states.
  </Tab>
  
  <Tab title="Accent Colors">
    **Accent Green** - Secondary green palette for highlights and accents
    
    Provides variation while maintaining brand consistency.
  </Tab>
  
  <Tab title="Extended Colors">
    Additional color palettes:
    - **Blue** - Information and links
    - **Yellow** - Warnings and highlights
    - **Orange** - Attention states
    - **Red** - Errors and critical actions
  </Tab>
  
  <Tab title="Neutral Colors">
    **Gray Scale** - From white to black with 10 shades
    
    Used for text, backgrounds, and UI elements.
  </Tab>
</Tabs>
