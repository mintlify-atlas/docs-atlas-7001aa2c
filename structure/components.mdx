## Overview

Components in Astro are reusable pieces of UI that can be imported and used across multiple pages. ZOODOM currently has one essential component: the **Head** component.

## Component Directory

```
src/components/
└── Head.astro    → SEO and meta tag management
```

## Head Component

The `Head.astro` component is responsible for managing all HTML `<head>` elements, including SEO metadata, social media tags, and global styles.

### Location

```
src/components/Head.astro
```

## Component Interface

The Head component accepts four optional props:

```typescript src/components/Head.astro
interface Props {
    title?: string;
    description?: string;
    canonicalUrl?: string;
    image?: string;
}
```

### Default Values

```typescript src/components/Head.astro
const {
    title = 'ZOODOM - Parque Zoológico Nacional',
    description = 'ZOODOM - Parque Zoológico Nacional. Descubre la fauna y flora en nuestro zoológico.',
    canonicalUrl = 'https://zoodomi.netlify.app',
    image = '/cover.png',
} = Astro.props;
```

<Info>
  All props have sensible defaults, so you can use the component without passing any props for standard pages.
</Info>

## Global CSS Import

The component imports the global stylesheet:

```astro src/components/Head.astro
---
import '../assets/global.css';
---
```

<Check>
  This ensures the design system and CSS variables are available throughout the entire application.
</Check>

## Social Image URL

The component generates a fully qualified URL for social media images:

```typescript src/components/Head.astro
const socialImageUrl = new URL(image, canonicalUrl).href;
```

This ensures the image URL is absolute, which is required by Open Graph protocols.

## Usage Example

### Default Usage

Use without props for default metadata:

```astro
---
import Head from '../components/Head.astro';
---

<html>
  <head>
    <Head />
  </head>
  <body>
    <!-- Page content -->
  </body>
</html>
```

### Custom Metadata

Override defaults with custom props:

```astro
---
import Head from '../components/Head.astro';
---

<html>
  <head>
    <Head 
      title="Especies - ZOODOM"
      description="Explora las diferentes especies de animales en nuestro zoológico"
      canonicalUrl="https://zoodomi.netlify.app/especies"
      image="/especies-cover.png"
    />
  </head>
  <body>
    <!-- Page content -->
  </body>
</html>
```

<Note>
  The Layout component already includes the Head component, so you typically don't need to import it directly in pages.
</Note>

## Creating New Components

To add more components to ZOODOM:

<Steps>
  <Step title="Create a new .astro file">
    Add your component in `src/components/`. For example, `Footer.astro`.
  </Step>
  
  <Step title="Define the component interface">
    ```astro
    ---
    interface Props {
      // Define your props here
    }
    const { /* props */ } = Astro.props;
    ---
    ```
  </Step>
  
  <Step title="Write the component markup">
    ```astro
    <footer>
      <!-- Your component HTML -->
    </footer>
    ```
  </Step>
  
  <Step title="Import and use in pages or layouts">
    ```astro
    ---
    import Footer from '../components/Footer.astro';
    ---
    
    <Footer />
    ```
  </Step>
</Steps>

## Component Best Practices

  <Card title="Single Responsibility" icon="bullseye">

  <Card title="Props with Defaults" icon="gear">

    Define Props interfaces for type safety and documentation
  </Card>
  
  <Card title="Reusability" icon="recycle">
