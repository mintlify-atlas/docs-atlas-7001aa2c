ZOODOM integrates **Bootstrap 5.3.8** via CDN to leverage a comprehensive component library while maintaining custom brand styling through CSS variables.

## Integration Method

Bootstrap is loaded in the main layout file (`src/layouts/Layout.astro`) using CDN links with Subresource Integrity (SRI) for security.

### CSS Integration

Bootstrap's CSS is loaded in the `<head>` section:

```html
<link
  href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css"
  rel="stylesheet"
  integrity="sha384-sRIl4kxILFvY47J16cr9ZwB07vP4J8+LH7qKQnuqkuIAvNWLzeN8tE5YBujZqJLB"
  crossorigin="anonymous"
/>
```

**Location**: `src/layouts/Layout.astro:9-14`

### JavaScript Integration

Bootstrap's JavaScript components are loaded at the end of the `<body>` for optimal performance:

```html
<!-- Popper.js - Required for dropdowns, popovers, and tooltips -->
```

## Security Features

### Subresource Integrity (SRI)

All Bootstrap files are loaded with SRI hashes to ensure file integrity:

- Prevents tampering with CDN resources
- Verifies that files haven't been modified
- Adds an extra layer of security

### Cross-Origin Resource Sharing (CORS)

The `crossorigin="anonymous"` attribute:

- Enables CORS requests
- Prevents sending user credentials
- Required for SRI to work properly

## CDN Benefits

### Performance

1. **Caching**: Users may already have Bootstrap cached from other sites
2. **CDN speed**: jsDelivr provides fast, geographically distributed servers
3. **Reduced server load**: Assets served from external CDN
4. **Parallel downloads**: Browser can download CDN assets simultaneously with your assets

### Maintenance

- No need to bundle Bootstrap in your build process
- Easy version updates (change version number in URL)
- Automatic minification and optimization

## Customization

### Using Bootstrap with Custom Colors

While Bootstrap provides its own color system, ZOODOM maintains custom brand colors through CSS variables. You can combine both:

```html
<!-- Bootstrap class with custom color -->
<button class="btn" style="background-color: var(--color-brand-green-600)">
  Custom Green Button
</button>

<!-- Or use Bootstrap's success color (green) -->
<button class="btn btn-success">
  Bootstrap Success Button
</button>
```

### Overriding Bootstrap Styles

Custom styles are loaded after Bootstrap, allowing overrides:

```css
/* In global.css */
.btn-success {
  background-color: var(--color-brand-green-600);
  border-color: var(--color-brand-green-600);
}

.btn-success:hover {
  background-color: var(--color-brand-green-700);
  border-color: var(--color-brand-green-700);
}
```

## Responsive Breakpoints

Bootstrap 5.3.8 uses these breakpoints:

| Breakpoint | Class Infix | Dimensions |
|------------|-------------|------------|
| Extra small | None | &lt;576px |
| Small | `sm` | ≥576px |
| Medium | `md` | ≥768px |
| Large | `lg` | ≥992px |
| Extra large | `xl` | ≥1200px |
| Extra extra large | `xxl` | ≥1400px |

## Best Practices

### 1. Use Bootstrap Classes First

Leverage Bootstrap's utility classes before writing custom CSS:

```html
<!-- Good: Using Bootstrap utilities -->
<div class="d-flex justify-content-between align-items-center p-4">

<!-- Avoid: Custom CSS for common layouts -->
<div style="display: flex; justify-content: space-between; align-items: center; padding: 1rem;">
```

### 2. Combine with Custom Colors

Use Bootstrap layout and ZOODOM colors:

```html
<div class="card" style="border-color: var(--color-brand-green-500)">
  <div class="card-body" style="background-color: var(--color-brand-green-50)">
    <h5 class="card-title" style="color: var(--color-brand-green-900)">Title</h5>
  </div>
</div>
```

### 3. Semantic HTML

Use semantic HTML elements with Bootstrap classes:

```html
<!-- Good -->
<nav class="navbar">
<header class="container">
<section class="row">

<!-- Avoid -->
<div class="navbar">
<div class="container">
```

### 4. Mobile-First Approach

Bootstrap is mobile-first, so structure your classes accordingly:

```html
<!-- Mobile-first: Start with mobile, add desktop classes -->
<div class="col-12 col-lg-6">
```

## Performance Considerations

### CDN vs. Self-Hosted

**Current (CDN):**
- ✓ Fast delivery via global CDN
- ✓ Potential browser cache hits
- ✓ No build process needed
- ✗ External dependency
- ✗ Requires internet connection

**Alternative (Self-hosted):**
- ✓ Full control over assets
- ✓ Works offline
- ✓ Can tree-shake unused CSS
- ✗ Larger bundle size
- ✗ Requires build process

### Optimization Tips

1. **Remove unused components**: Consider using Bootstrap's Sass source to include only needed components
2. **Defer non-critical JavaScript**: Add `defer` attribute if interactive components aren't immediately needed
3. **Consider bundle**: The bundle version combines Bootstrap + Popper, reducing HTTP requests

## Migration to Self-Hosted

If you want to self-host Bootstrap in the future:

```bash
npm install bootstrap @popperjs/core
```

Then import in your Astro layout:

```astro
---
import 'bootstrap/dist/css/bootstrap.min.css';
---
```

And add to your Astro config:

```javascript
// astro.config.mjs
export default {
  vite: {
    resolve: {
      alias: {
        '~bootstrap': path.resolve(__dirname, 'node_modules/bootstrap'),
      }
    }
  }
}
```
